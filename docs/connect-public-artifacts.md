# Skipper Connect: public artifact links (integrator spec)

The daemon can publish an artifact version for public viewing. The public URL is served by the connect integrator, which relays the request to the owning daemon over its existing authenticated WebSocket. This document specifies the route the integrator must implement.

## URL shape

```
GET /p/:guid/:artifactId?key=<publishKey>
```

- `:guid` is the instance global id (the `gid` claim of the daemon's connect key) identifying which connected daemon owns the artifact. The daemon derives it from the key; it is not separately configured.
- `:artifactId` is the `task_artifacts.id` UUID of one specific artifact version.
- `key` is the per-version publish key generated by the daemon on first publish. It is stable for the lifetime of that artifact version.

The route is unauthenticated. Possession of the full URL is the credential; the daemon validates the key.

## Handling

1. Look up the live WebSocket connection for `:guid`. If that instance is not connected, respond `404` (or a friendly "instance offline" page). Do not queue.
2. Send a resource request over the socket:

   ```json
   { "type": "request", "id": "<uuid>", "resource": "artifacts", "action": "read-published", "params": { "id": "<artifactId>", "key": "<key>" } }
   ```

3. Await the matching `{ "type": "response", "id": ... }`. Apply a timeout of about 10 seconds; on timeout respond `504`.
4. If `ok` is `false`, respond `404` with a generic body. The daemon returns one opaque error for wrong id, wrong key, and unpublished; the integrator must not add detail.
5. If `ok` is `true`, `data` contains:

   ```json
   { "name": "...", "kind": "...", "version": 3, "body": "...", "contentType": "text/html; charset=utf-8" }
   ```

   Respond `200` with `data.body` as the response body and headers:

   - `Content-Type: <data.contentType>` (either `text/html; charset=utf-8` or `text/plain; charset=utf-8`)
   - `X-Robots-Tag: noindex`
   - `Cache-Control: no-store` (unpublish must take effect immediately; never cache bodies)
   - `Content-Security-Policy: sandbox` (blocks scripts and gives the document a unique origin, so published HTML cannot ride an operator's integrator session cookie into the authenticated API)
   - `Referrer-Policy: no-referrer` (the publish key is in the query string; this guarantees links inside published HTML never leak it via a Referer header)

Implemented in the skipper-connect worker (`src/index.ts`, route `GET /p/:gid/:artifactId`), alongside integrator REST endpoints `POST /api/v1/artifacts/:id/publish` and `POST /api/v1/artifacts/:id/unpublish` gated by the `artifacts:publish` scope.

## Daemon-side behavior (for reference)

- Publish state lives on `task_artifacts` (`publish_key`, `published_at`). Publishing is per version; other versions of the same artifact name stay private.
- `publish_key` is generated once per version and never rotated. Unpublish sets `published_at` to NULL; the same URL works again after republish.
- The key comparison is constant-time and the daemon never distinguishes missing vs unpublished vs wrong key.
- Publish/unpublish are triggered from the Skipper UI artifact modal, or remotely via the `artifacts` resource actions `publish` / `unpublish` (params `{ id }` or `{ taskId, name, version? }`), which return `publicUrl`.
